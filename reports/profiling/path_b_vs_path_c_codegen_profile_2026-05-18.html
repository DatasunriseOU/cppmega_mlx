Verdict

What actually broke and what changed

Mamba3 bwd C/B1.36x -> 1.05x

Hot-loop allreduce removed.

M2RNN fused fwd+bwd C/B2.08x -> 1.12x

Recompute-bwd removed from production VJP.

Path B is 72.12 MB on the same receipt.

Mamba3 fwd C/B1.32x

Still a forward codegen/schedule gap.

Generic rule: recurrent generated bwd must not put cross-lane final-gradient reductions or feature-wise scan recomputation inside the time/state hot loop. Mamba3 now emits partial gradients and reduces outside the scan kernel; M2RNN fused VJP now reuses the generated recurrent output and ordinary generated post-bwd, while recurrent backward remains the reduced Path B kernel in production.

What We Compare

Scope and baselines

  • Path B: hand-written MLX Metal kernels and existing Path B VJP behavior.
  • Path C: TileLang DSL lowered through TVM and launched through tvm-ffi owner-output kernels.
  • Mamba3 shape: B=1, T=2048, H=112, P=64, N=64, dtype=bfloat16.
  • M2RNN shape: B=1, T=2048, total_heads=4, K=64, V=16, dtype=bfloat16.
  • Training matrix: full 1B bf16/fp8 optimizer matrix is in venv_fixed_8bit_matrix_2026-05-17.html. These model-shape benches isolate the generated-bwd codegen problem found from that matrix.

Similar Cases Checked

Generated bwd scan

  • Sparse-MLA BF16 Path C bwd: direct tvm-ffi owner-output path uses atomic dKV accumulation; tests guard against public partial dKV tensors. No Mamba-style hot-loop thread_allreduce pattern found.
  • Sparse-MLA FP8 Path C bwd: direct tvm-ffi owner-output path has explicit dq/dkv owner buffers and clear kernel; tests guard that retired direct-MSL partial routes stay removed.

Changed Code

Generic fixes, not shape-specific patches

  • cppmega_mlx/nn/_tilelang/mamba3_path_c.py: added the generic partial-output bwd route for Mamba3 and made public Path C bwd dispatch through it.
  • tests/test_tilelang_mamba3_path_c.py: checks lowered kernel bodies for absence of hot-loop allreduce/barriers and covers unaligned plus aligned headdims.
  • cppmega_mlx/nn/_tilelang/m2rnn_path_c.py: changed mapped-packed-post VJP to use generated recurrent output plus ordinary post-bwd; recompute-bwd remains diagnostic only.

  • scripts/bench_tilelang_m2rnn_path_c.py: labels recompute post-bwd as diagnostic in report MSL metrics.
