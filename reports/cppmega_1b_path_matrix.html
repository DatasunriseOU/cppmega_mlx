Measurements

Route Definitions

The dtype label and path label both matter.

BF16 Path B

Runs --dtype bfloat16 with CPPMEGA_KERNEL_PATH=auto. This is the current non-forced baseline route. It may use existing Path B or reference surfaces according to the normal runtime policy.

BF16 Path C

Runs --dtype bfloat16 with CPPMEGA_KERNEL_PATH=path_c. Cold rows clear/use an empty TileLang cache; warm rows reuse the per-dtype optimizer TileLang cache.

FP8 Path B

Runs --dtype fp8_path_b and forces CPPMEGA_KERNEL_PATH__SPARSE_MLA=path_b. The DSA Sparse-MLA baseline dispatch is recorded as sparse_mla_fp8_reference_path_b; it is an honest non-Path-C FP8 training baseline, not a Path C fallback.

FP8 Path C

Runs --dtype fp8_path_c and forces Path C for Mamba3, M2RNN, and Sparse-MLA. Sparse-MLA consumes prepared q_fp8/q_scale/kv_fp8/kv_scale buffers through the TileLang/tvm-ffi route.
