### **Design/Architecture findings**

**1. Violated abstractions / Leaky modules**  
`sparse_mla_path_c.py` contains 20+ `_canonicalize_*` functions. These are MSL text transforms. Abstraction leak: kernel file is doing compiler-pass work. Should be in `_msl_transform.py` or `tilelang_lowering/`. Path B doesn’t do this.

**2. Duplicated logic**  
Both paths implement VJP: `_sparse_mla_metal_apply_vjp` vs `_sparse_mla_path_c_metal_apply_vjp`. Same `_reference_apply` pattern. Should share a `make_vjp` helper.

**3. Awkward coupling**  
Path C status depends on tilelang availability `_tilelang_available()`. Path B checks MLX Metal `sparse_mla_metal_status`. Two different feature gates for same op. Docs say env var `CPPMEGA_KERNEL_PATH=path_c` redirects but that couples runtime selection to global env, not per-call args.

**4. Missing extension points**  
Path C adds `mask_value` but Path B has it hardcoded. If you need different mask, you’re forced to Path C. No shared config struct. Suggest: `SparseMLAConfig(mask_value=..., sm_scale=...)` passed to both.

**5. Naming/clarity**  
`_sparse_mla_path_c_apply_for_params` returns a closure. Name suggests it’s for params but it binds `sm_scale, d_v`. Unclear vs `sparse_mla_path_c_apply` which is the public API. Path B uses `sparse_mla_apply`. Inconsistent suffix: `_apply` vs `_path_c_apply`. Pick one convention.

**6. Over-engineering**  
2506 LOC for one kernel path. Most is MSL string munging. If TileLang can’t emit clean MSL, fix TileLang or add a single `tilelang_msl_postpass.py`. Don’t copy 30 regex transforms per kernel. `_remove_redundant_flat_kv_bounds_checks` and friends should be table-driven, not per-file.

**7. Under-engineering**  
`_path_b_lowering.py` exists but unused. If Path B needs lowering, it should use it. If not, delete. Docs claim “Path B carries sequence-dimension reductions” but code structure doesn’t show where.

### **Actionable suggestions to simplify**

1. **Extract MSL postprocess**: Move all `_canonicalize_*`, `_insert_*`, `_drop_unused_*` from `sparse_mla_path_c.py` into `_msl_transform.py` as `apply_path_c_passes(msl, shape_config)`. Kernel files should only define `def sparse_mla_fwd():` TileLang body. LOC drop ~1500.

2. **Unify VJP**: Create `_make_sparse_mla_vjp(fwd_fn, bwd_fn)` in shared `_autograd.py`. Both paths call it. Deletes 40 LOC dup.

3. **Single feature gate**: Replace `sparse_mla_metal_status` + `sparse_mla_path_c_status` with `kernel_capabilities(op="sparse_mla", backend="path_b"|"path_c")`. Routing doc then matches code.

4. **Config object**: `SparseMLA(sm_scale=1.0, d_v=64, mask_value=-65504.0).apply(q,kv,idx)`. Dispatch inside based on env/shape. Removes signature drift.
