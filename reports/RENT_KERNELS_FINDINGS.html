🎯 The headline finding

DeepSeek V4 exists (4 HF repos, 22 April 2026, 3.1M downloads on V4-Pro) and ships with TileKernels — a pure-TileLang kernel layer covering MoE, Engram, mHC (Manifold HyperConnection), and Quant. Two of our existing blocks (MHC and N/Engram) have direct DeepSeek equivalents in TileKernels.

For GatedDeltaNet, mlx-lm PR #1217 already provides a Metal fwd+bwd kernel with LoRA-converged training. Cherry-pick that and wrap as our L block.

2.1 Module layout

moe/ (12 files)

Top-k gating, fused expand/reduce, normalize, aux-loss-free routing. Key files: aux_fi_kernel.py, topk_gate_kernel.py, top2_sum_gate_kernel.py, expand_to_fused_kernel.py, reduce_fused_kernel.py, mask_indices_by_tp_kernel.py, normalize_weight_kernel.py, scoring.py.

quant/ (FP8/FP4/E5M6)

Per-token / per-block / per-channel quantization. Fused SwiGLU + quantize patterns.

engram/ (4 files)

engram_fused_weight_kernel.py, engram_gate_kernel.py, engram_grad_w_reduce_kernel.py, engram_hash_kernel.py. Fused RMSNorm + gate + grad-reduce + hash routing.

mhc/ (9 files)

sinkhorn_kernel.py + 8 supporting kernels (expand, head_compute_mix, multilayer_recompute, norm_fn, post, pre_apply_mix, pre_big_fuse, pre_split_mixes). Manifold HyperConnection with Sinkhorn-Knopp Birkhoff projection.

torch/ (golden)

Pure-PyTorch reference impls. This is what we port to MLX first for correctness baselines.

modeling/

torch.autograd.Function wrappers: engram_gate, mhc_pipeline. The Python-side glue.

2.2 mHC forward math

X_{l+1} = B_l · X_l + C_l · F_l(A_l · X_l)

  where  X_l shape = (B, S, n=4, D)        — 4 parallel residual streams
         B_l        — Birkhoff polytope, Sinkhorn-Knopp normalized (20 iters)
         A_l, C_l   — learnable mix-split / mix-apply matrices
         F_l        — block's residual function (attention or FFN)
⚠ Sinkhorn backward gotcha: The naive backward assumes r=c=1 which only holds at full convergence. Use the corrected version from TileKernels Issue #2 — it solves (diag(c) − Rᵀ·diag(r)⁻¹·R)·β = s_c − Rᵀ·diag(r)⁻¹·s_r via two matvecs, never materializing RᵀR.

3. GatedDeltaNet — implementation landscape

3.1 The recurrence

S_t = α_t · (I − β_t · k_t · k_tᵀ) · S_{t-1} + β_t · k_t · v_tᵀ
y_t = S_tᵀ · q_t

Chunkwise parallel form: 4 phases per chunk — intra-chunk Householder prefix products, inter-chunk state recurrence, intra-chunk output, mirrored backward with solve_tril.

3.1.5 Two separate blocks: "L" (GDN) and "K" (KDA)

🧩 Why two blocks, not one parametrized?

GDN (classical Householder recurrence) and KDA (DPLR super-set with gated-LA branch) differ in state shape, recurrence inner loop, and test surface. Two distinct blocks in the pattern is cleaner — symbol L for GDN, symbol K for KDA.

🔹 Symbol "L" — GatedDeltaNet (5 paths)

MLX + Metal MIT ~5-6 days (all paths)

Recurrence: S = α(I − β·k·kᵀ)·S + β·k·vᵀ. Land all 5 paths, then auto-mode picks winner per shape.

  • Path A pure-MLX from FLA math (golden, ~1d)
  • Path B hand-written MSL via mx.fast.metal_kernel (~1d)
  • Path C TileLang DSL via tilelang.compile(target="metal") from examples/gdn/ (~1.5d)
  • Path D Triton frontend lift of FLA Triton kernel (~0.5d)
  • Path E vendor mlx-lm PR #1217 (~1d)
  • HF parity: m-a-p/1.3B-100B-GatedDeltaNet-pure

🔸 Symbol "K" — Kimi Delta Attention (4 paths)

TileLang + Metal ~4-5 days (all paths)

DPLR: A = αI + β·u·vᵀ. Strict super-set of GDN with gated-LA branch. +2-3% perplexity vs GDN. Already in vLLM.

  • Path A pure-MLX from FLA_KDA (~1d)
  • Path B hand-MSL (~1d)
  • Path C TileLang DSL from examples/kda/ (11 prims, ~1.5d)
  • Path D Triton frontend (~0.5d)
  • No Path E — no mlx-lm KDA op exists yet

Mirror mamba3_path_c_receipt_auto_mode machinery: per-(block, dtype, shape) JSON receipt drives auto_mode_for_inputs dispatch. HTML grid like cppmega_1b_path_matrix.html.

5. Key documents and gotchas

📄 FlashMLA absorb trick

Folds W_UK · W_O into the QK GEMM. Halves decode FLOPs without changing output. Apple Silicon: port the algebra, skip the Hopper kernel.

~/sources/rent_kernels/FlashMLA/docs/20250422-new-kernel-deep-dive.md

📄 FlashMLA FP8 sparse decoding

410 TFLOPS on H800, 350 TFlops on B200. Pipeline + scaling strategy doc.

~/sources/rent_kernels/FlashMLA/docs/20250929-hopper-fp8-sparse-deep-dive.md

📄 GDN Rosetta stone

Full math derivation, gotcha S@k vs Sᵀ@k, alpha-gate fusion, state quantization, per-variant grid/block/LDS table.

~/sources/rent_kernels/hipfire/DELTANET.md

⚠ Lightning Indexer RoPE gotcha

Indexer Q/K use non-interleaved RoPE. MLA Q/K use interleaved RoPE. Mixing these silently produces wrong indices. V3.2-Exp README flags this explicitly.

⚠ Sinkhorn corrected backward

Naive backward assumes r=c=1 (only true at full convergence). Use TileKernels Issue #2 corrected formula — two matvecs, never materialize RᵀR.

📂 V3.2 indexer split across 3 repos

Sparse attention kernels in FlashMLA/csrc/; Lightning Indexer logit kernels in DeepGEMM/deep_gemm/include/; readable TileLang reference at tilelang/examples/deepseek_v32/sparse_mla_fwd_seesaw.py.

6. Apple Silicon vs CUDA strategy

🍎 Apple Silicon (cppmega.mlx)

Apple Silicon = SM120 desktop Blackwell (no TMEM, no tcgen05). Mirror SGLang PR #24692:

  • Don't lift TileLang kernels directly
  • Use TileKernels torch/ golden as MLX correctness baseline
  • Write Metal-shader fallbacks with same fused patterns
  • Engram is "free": unified memory natively covers the HBM↔DRAM split

🚀 CUDA (cppmega for GB10/H200/B200)

Wire TileKernels via Megatron ModuleSpec machinery:

  • H200: TileKernels SM90 build (WGMMA FP8, FP32 scales)
  • B200/GB10: SM100 build (TMEM, UE8M0 packed scales, FP4 weights)
  • GB10 unique: Grace LPDDR5X is natural Engram backing store via C2C coherent
