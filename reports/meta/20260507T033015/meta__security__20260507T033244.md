### NEW/MISSED Security Issues Introduced or Worsened

#### 1. **Arbitrary Code Execution via Insecure Library Preload Path** `HIGH`
**Location**: `_msl_transform.py:88`, `_msl_transform.py:102`  
**Issue**: Preload candidate includes world-writable `/tmp/tl_apache_tvm_swap/build/lib/libz3.dylib`.  
```python
  88 | candidates.append(_Path("/tmp/tl_apache_tvm_swap/build/lib/libz3.dylib"))
```
**Attacker scenario**: Any local user or compromised process can drop a malicious `libz3.dylib` in `/tmp/...` before Python starts. `ctypes.CDLL` at `_msl_transform.py:98` will `dlopen` it with the privileges of the running Python process. On macOS this executes attacker code at import time, before any user auth.  
**Impact**: Local privilege escalation to the Python process user. In CI runners or multi-tenant notebooks, this is RCE.  
**Fix**: Remove `/tmp` from candidates. Only load from the package’s own `site-packages` or signed wheels. If dev fallback is required, gate behind `CPPEMEGA_ALLOW_UNSAFE_LIBZ3=1` env var and log a warning.

#### 2. **TOCTOU in libz3 Preload Allows Race to Swap Library** `MEDIUM`
**Location**: `_msl_transform.py:94-97`  
**Issue**: Code checks `candidate.exists()` then `dlopen` in separate syscalls. An attacker with write access to the directory can race: create valid libz3 after `exists()` but before `dlopen`, or swap symlink to malicious dylib.  
**Attacker scenario**: In `/opt/homebrew/lib` this is unlikely, but in user-controlled build dirs or `/tmp`, a race wins RCE.  
**Fix**: Use `ctypes.util.find_library` or attempt `dlopen` directly and catch `OSError` instead of pre-checking `exists()`. Remove the race window.

#### 3. **Path Traversal in Build Root Derivation** `MEDIUM`
**Location**: `_msl_transform.py:85-86`  
**Issue**: `build_root_from_cwd()` walks up parents looking for `build/libz3.dylib` `_msl_transform.py:86`. If the process CWD is attacker-controlled, `Path.cwd()` may resolve inside `/tmp/attacker`, causing preload from arbitrary location.  
**Attacker scenario**: Web service that changes CWD per request or Jupyter notebook started in `/tmp/uploads/user123`. Attacker uploads `build/lib/libz3.dylib` and triggers import.  
**Fix**: Do not derive search paths from `CWD`. Use only `__file__`-relative paths or package resources.

#### 4. **Missing Input Validation on Kernel Template Injection** `LOW-MEDIUM`
**Location**: `_msl_transform.py:8296-8315`, `topk_selector.py:2232-2240`  
**Issue**: `template=[("typename K", K), ("typename K_DOUBLE", K_DOUBLE)]` is passed to `T.metal`. If `K` or `K_DOUBLE` originates from user-controlled shapes, large integers could cause Metal shader compilation to exhaust memory or time. No clamp.  
`_msl_transform.py:8121-8122` comment admits MSL lacks arithmetic on template constants, so you do it in Python.  
**Attacker scenario**: Call `topk_selector(scores, k=1<<30)` from untrusted input. Kernel launch OOMs the GPU or hangs the host in shader compile.  
**Fix**: `assert 1 <= K <= 65536` and similar bounds before creating `template`. Reject non-ints.

#### 5. **Unsafe Deserialization Surface via TileLang/TVM** `INFO`  
**Location**: `_msl_transform.py:60-61`, `_msl_transform.py:1861-1862`  
**Issue**: Comments acknowledge running without TVM/libz3 in CI. If TVM is present, TileLang kernels may be cached as pickles or IRModule blobs. No evidence of `pickle.load` on untrusted data in this diff, but the preload code makes TVM importable. If upstream TileLang later loads user-provided `.tl` files, `eval` in `T.evaluate(` appears at `_msl_transform.py:407`, `_msl_transform.py:500`.  
**Action**: Add allowlist for `.tl` sources. Ensure `T.evaluate` only receives constants, not strings from metadata.
