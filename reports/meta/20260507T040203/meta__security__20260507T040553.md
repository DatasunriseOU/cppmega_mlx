### Critical Findings

#### 1. Arbitrary Code Execution via World-Writable `/tmp` dylib Load  
**Severity: Critical**  
**Location: `cppmega_mlx/nn/_tilelang/_msl_transform.py:1304-1325`**  

```python
1314 |  98 |     if os.environ.get("CPPMEGA_ALLOW_UNSAFE_LIBZ3") == "1":
...
1325 | 109 |         candidates.append(_Path("/tmp/tl_apache_tvm_swap/build/lib/libz3.dylib"))
...
1332 | 116 |                 ctypes.CDLL(str(candidate), ctypes.RTLD_GLOBAL)
```

**Why it’s exploitable**  
1. `/tmp` is world-writable on macOS/Linux. Any local unprivileged user can create `/tmp/tl_apache_tvm_swap/build/lib/libz3.dylib` before your process starts.
2. If `CPPMEGA_ALLOW_UNSAFE_LIBZ3=1` is set, the helper unconditionally `dlopen`s that path via `ctypes.CDLL`, loading attacker-controlled native code into the Python process with full process privileges.
3. The comments admit this: `security HIGH: /tmp is world-writable; an attacker who can write there could plant a malicious libz3.dylib that gets dlopen'd into the process` `cppmega_mlx/nn/_tilelang/_msl_transform.py:1305`.

**Concrete attacker scenario**  
1. Victim runs any script importing `cppmega_mlx.nn._tilelang._msl_transform` on a multi-user macOS dev box or CI runner.  
2. Attacker, as `nobody`, pre-creates `/tmp/tl_apache_tvm_swap/build/lib/libz3.dylib` containing a `__attribute__((constructor))` that spawns a reverse shell.  
3. Victim sets `CPPMEGA_ALLOW_UNSAFE_LIBZ3=1` for a bench run. On import, `ctypes.CDLL` executes attacker code as the victim. If the victim is root or a CI job with cloud creds, it’s a full takeover.

**Additional factors**  
1. TOCTOU race `cppmega_mlx/nn/_tilelang/_msl_transform.py:1307-1313`: `exists()` then `dlopen()` lets an attacker swap the file after the check. Comment acknowledges this but accepts it.  
2. Env-var gate is not checked for privilege: if a setuid Python wrapper ever imports this module with user-controlled env, it’s root RCE.  

**Remediation**  
1. Drop `/tmp` from `candidates` entirely. Require `libz3.dylib` to live in root-owned paths: `/opt/homebrew/lib`, `sys.prefix/lib`, or a path explicitly passed via `TILELANG_DEV_BUILD_ROOT`.  
2. If `/tmp` must stay for dev, add: `os.stat()` check for `st_uid == 0` and `not stat.S_IWOTH & mode`. Abort if world-writable.  
3. Never honor `CPPMEGA_ALLOW_UNSAFE_LIBZ3` when `os.geteuid() == 0` or in production builds. Fail closed.  
4. Consider `ctypes.util.find_library` or `importlib.metadata` instead of hard paths.

#### 2. Privilege Escalation via Tainted Env-Var Controlling Code Path  
**Severity: High**  
**Location: `cppmega_mlx/nn/_tilelang/_msl_transform.py:1314`, `tests/conftest.py:8959`, `tests/conftest.py:9022`**  

The gate `CPPMEGA_ALLOW_UNSAFE_LIBZ3` directly enables the `/tmp` loading behavior. Tests set it unconditionally:  
`tests/conftest.py:8959`: `os.environ.setdefault("CPPMEGA_ALLOW_UNSAFE_LIBZ3", "1")`  
`tests/conftest.py:9022`: `monkeypatch.setenv("CPPMEGA_ALLOW_UNSAFE_LIBZ3", "1")`  

**Attacker scenario**: Any code path that can set environment variables before Python starts can flip this to unsafe. In CI, a compromised dependency could export the var in a `setup.py` or `pytest` plugin, silently enabling the `/tmp` RCE above for all subsequent test runs.  

**Remediation**: Rename to `CPPMEGA_DEV_ALLOW_UNSAFE_LIBZ3` and add `if "CI" in os.environ or sys.flags.optimize > 0: raise RuntimeError`. Never set in test fixtures; require explicit CLI flag.

### High Findings

#### 3. TOCTOU Race in `libz3` Preload Allows File Replacement  
**Severity: High**  
**Location: `cppmega_mlx/nn/_tilelang/_msl_transform.py:1331-1339`**  

```python
1331 | 115 |             if candidate.exists():
1332 | 116 |                 ctypes.CDLL(str(candidate), ctypes.RTLD_GLOBAL)
1333 | 117 |                 _preload_libz3_for_dev_tilelang._done = True
...
1336 | 120 |         except FileNotFoundError:
1337 | 121 |             # fix-round-5: TOCTOU — file vanished between exists() and dlopen.
```

Even without `/tmp`, an attacker with write access to `TILELANG_DEV_BUILD_ROOT` can race: create a benign `libz3.dylib`, let `exists()` pass, then replace with malicious dylib before `dlopen`. The window is small but real on shared filesystems or containers with mounted volumes.  

**Remediation**: Use `os.open(path, os.O_RDONLY)` then `ctypes.CDLL(fd)` via `/proc/self/fd/N` on Linux. On macOS, no fd-dlopen exists, so require the directory to be non-writable by others: `os.stat(parent).st_mode & stat.S_IWOTH == 0`.

#### 4. Potential Path Traversal in FFI Image Enumeration  
**Severity: Medium**  
**Location: `cppmega_mlx/nn/_tilelang/_msl_transform.py:1550-1552`**  

```python
1550 | 334 |         path = os.path.realpath(raw.decode("utf-8", errors="replace"))
1551 | 335 |         if os.path.basename(path) != "libtvm_ffi.dylib" or path in seen:
```

`raw` comes from `ctypes._dyld_shared_cache_contains_path` / `image_name(idx)`, i.e., dyld’s list of loaded images. Normally safe. Risk: if Python is embedded in an app where an attacker controls `DYLD_INSERT_LIBRARIES`, they can inject an image with a crafted path containing `../` that still ends in `libtvm_ffi.dylib`. `realpath` resolves it, but the resolved path is then trusted for later logic. No current sink, but if this list is ever used to `dlopen` or read files, it becomes LFI.  

**Remediation**: After `realpath`, assert `path.startswith(allowed_prefix)` for known install dirs. Don’t trust dyld output for security decisions.

#### 6. Metal Kernel Names Built from Integer Dimensions Without Sanitization  
**Severity: Low**  
**Location: `cppmega_mlx/nn/_tilelang/sparse_mla_path_c.py:6989`, `7210`**  

```python
6989 | 1947 |             f"cppmega_sparse_mla_path_c_fwd_noguard_"
6990 | 1948 |             f"{BATCH}_{SEQ_LEN}_{HEADS}_{QK_DIM}_{KV_GROUP}_{TOPK}_{SEQ_LEN_KV}_{D_V}_{THREADS}"
```

All interpolated values are `int` from tensor shapes. If a future refactor accidentally passes a string, it could inject Metal source. Today it’s safe because `mx.fast.metal_kernel` treats `name` as identifier only, not code.  

**Remediation**: `f"..._{int(BATCH)}_..."` or validate `isinstance(..., int)` before formatting. Defense-in-depth.

### Performance / Security Adjacent

#### 7. Repeated `os.environ.get` in Hot Path  
**Location: `cppmega_mlx/nn/_tilelang/fp8_vecmat_path_c.py:4501`**  
`debug = os.environ.get("CPPMEGA_FP8_VECMAT_PATH_C_DEBUG", "0")` is called inside `make_fp8_vecmat_reduce_kernel`. For M=1 vecmat this is hot. Env lookups are not free and prevent `lru_cache` hits if wrapped.  

**Remediation**: Read once at module import: `_FP8_DEBUG = os.environ.get(...) != "0"`.
