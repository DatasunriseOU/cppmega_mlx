### 1. Leaky abstraction: dynamic loader policy embedded in transform layer
**`/Volumes/external/sources/cppmega.mlx/cppmega_mlx/nn/_tilelang/_msl_transform.py:50-145`**  
`_preload_libz3_for_dev_tilelang` lives inside the MSL transform module. This couples three unrelated concerns:

| Concern | Where it belongs |
| --- | --- |
| TileLang→MSL string rewrite | `_msl_transform.py` - correct |
| Darwin dyld workaround for dev builds | Platform bootstrap / `__init__` or dedicated `tilelang_compat.py` |
| Security policy for `/tmp` loads | Config/policy layer, not codegen |

**Bug**: Importing `_msl_transform.py` for any reason triggers `ctypes.CDLL` side effects on Darwin `cppmega_mlx/nn/_tilelang/_msl_transform.py:1360-1361`. A unit test that just wants to mock `lower_tilelang_to_msl_inline` will dlopen libz3. 

**Simplify**: Move preload to `cppmega_mlx/nn/_tilelang/__init__.py` behind `if sys.platform == "darwin"` and make it lazy. Better: remove it entirely and fix the tilelang build to use `@rpath`. The comment `cppmega_mlx/nn/_tilelang/_msl_transform.py:1269-1271` admits this is papering over a build bug.

### 2. Security gate is opt-out in tests, opt-in in prod — backwards
**`/Volumes/external/sources/cppmega.mlx/tests/conftest.py:8950-8959`**  
Tests force `CPPMEGA_ALLOW_UNSAFE_LIBZ3=1` globally at import time. Production defaults to safe. That means your CI never exercises the safe path. You shipped code where the *unsafe* path is the only one tested.

**Awkward coupling**: The env var is read in 2 places: `cppmega_mlx/nn/_tilelang/_msl_transform.py:1314` and `tests/conftest.py:8959`. If you rename it, tests silently diverge. 

**Fix**: Delete the conftest setenv. Add an explicit test `test_libz3_preload_skips_tmp_when_gate_off`. Make `/tmp` candidate append conditional on the env var, not gated after the fact.

### 3. Exception handling splits `FileNotFoundError` vs `OSError` but TOCTOU remains
**`cppmega_mlx/nn/_tilelang/_msl_transform.py:1336-1348`**  
You catch `FileNotFoundError` for “file vanished between exists() and dlopen” and `OSError` for “broken dylib”. But the comment `cppmega_mlx/nn/_tilelang/_msl_transform.py:1307-1313` admits the TOCTOU window exists and you’re not fixing it because macOS lacks fd-based dlopen.

**Design issue**: You’re checking `candidate.exists()` `cppmega_mlx/nn/_tilelang/_msl_transform.py:1331` then doing `ctypes.CDLL` anyway. The `exists()` call adds nothing except a race. If dlopen fails, it raises `OSError` regardless. The split handling is over-engineering.

**Simplify**: Drop the `exists()` check. Just try `CDLL` and catch `OSError`. If you need to distinguish “missing” vs “corrupt”, parse `e.args[0]` for `dlopen(...): image not found`. One try/except, no TOCTOU comment needed.

### 4. Extension point missing: candidate list is hard-coded
**`cppmega_mlx/nn/_tilelang/_msl_transform.py:1293-1327`**  
Candidates: `TILELANG_DEV_BUILD_ROOT`, `TILELANG_ROOT/build/lib`, `/tmp/...`, `/opt/homebrew/lib`. All hard-coded. If a user installs tilelang via conda in `~/mamba/envs/.../lib`, you won’t find it.

**Under-engineering**: No way to extend without patching the file. Yet you already parse env vars. 

**Fix**: Collapse to one env var `TILELANG_LIBZ3_SEARCH_PATH` with `os.pathsep`-delimited list. Pre-populate default from current heuristics. Now it’s testable and users can fix their own envs. Drop the special-case `/tmp` block — if a user *really* needs `/tmp`, they put it in the search path and take responsibility.

### 5. Violated abstraction: Z3 concepts leak into MSL string munging
**`cppmega_mlx/nn/_tilelang/_msl_transform.py:5166-5167, 5213-5217`**  
Function name `_strip_z3_hoisted_address_decls` and comments say “z3-final’s algebraic hoists”. But this module claims `cppmega_mlx/nn/_tilelang/_msl_transform.py:1240-1243` that “none of the Z3 roadmap proofs operate at this layer”. 

**Leaky module**: You’re doing AST-aware fixups in a regex-based string transform because the upstream passes emitted foot-guns. That’s an abstraction violation. The proper fix is a TVM TIR pass that guarantees the MSL you receive is already safe to inline.

**Naming/clarity**: If you must keep it, rename. `_strip_z3_hoisted_address_decls` implies you understand Z3 internals. You don’t — you’re pattern-matching `int h = ...`. Call it `_drop_pre_marker_hoisted_ints` and delete the Z3 editorial.

### 6. Duplicated logic: `failed_attempts` counter vs `_done` flag
**`cppmega_mlx/nn/_tilelang/_msl_transform.py:1283-1291, 1354`**  
You have both `_done` and `_failed_attempts`. `cppmega_mlx/nn/_tilelang/_msl_transform.py:1288-1291` bails after 3 failures. But if dlopen succeeds you set `_done=True` `cppmega_mlx/nn/_tilelang/_msl_transform.py:1334` and never touch `_failed_attempts` again.

**Awkward coupling**: The two attrs are never reset together. If env changes between calls, `_done` blocks retry but `_failed_attempts` could still be < 3. 

**Simplify**: One state: `state: Literal["init","loaded","give_up"]`. Delete the counter. If you want a retry cap, do it with `@lru_cache` or module-level boolean. You’re not in a hot loop.

### 7. Over-engineering: `pass_configs` plumbing for Z3 that doesn’t exist yet
**`cppmega_mlx/nn/_tilelang/_msl_transform.py:1777-1781`**  
Docstring advertises `pass_configs` to opt into Z3 roadmap passes. But `cppmega_mlx/nn/_tilelang/sparse_mla_path_c.py:3978-3983` says only a subset are registered and you silently drop unsupported keys.

**Missing extension point**: No way for a caller to query “which keys are supported in this libtilelang build”. They have to try and pray. That makes the API dishonest.

**Fix**: Add `lower_tilelang_to_msl_inline.supported_pass_configs() -> set[str]` that probes once. Or delete the param until the Z3 passes actually ship. Right now you’re adding surface area for zero benefit.

### 8. Performance: repeated stat() calls in candidate loop
**`cppmega_mlx/nn/_tilelang/_msl_transform.py:1329-1332`**  
For each candidate you do `candidate.exists()` which is a `stat`. If you have 4 candidates and libz3 is missing, you stat 4 files on every Path C dispatch until `_failed_attempts` hits 3. `cppmega_mlx/nn/_tilelang/_msl_transform.py:1285-1287` comment admits this.

**Bug**: On NFS or slow filesystems this is measurable. You already bail after 3 full sweeps, but that’s still 12 stats in the worst case.

**Simplify**: Per point 3, delete `exists()`. Or cache negative results: `if candidate in _MISSING: continue`. One line, no more stats.

**`cppmega_mlx/nn/_tilelang/_msl_transform.py:1238, 1777`**, **`cppmega_mlx/nn/_tilelang/sparse_mla_path_c.py:3978`**
